# 如何运行攻击模拟脚本

## 在 Linux/Mac 系统上运行

### 方法1：直接使用 bash 命令（最简单）
```bash
bash attack_backconnect.sh
bash attack_ngrok_ultra.sh
bash attack_mining_pools_fast.sh
```

### 方法2：添加执行权限后运行
```bash
# 添加执行权限
chmod +x attack_backconnect.sh
chmod +x attack_ngrok_ultra.sh
chmod +x attack_mining_pools_fast.sh

# 运行脚本
./attack_backconnect.sh
./attack_ngrok_ultra.sh
./attack_mining_pools_fast.sh
```

## 在 Windows 系统上运行

### 方法1：使用 WSL (Windows Subsystem for Linux)
```bash
# 在 WSL 终端中
cd /mnt/c/Users/14248/Desktop/EDR/rules
bash attack_backconnect.sh
```

### 方法2：使用 Git Bash
```bash
# 在 Git Bash 中
cd /c/Users/14248/Desktop/EDR/rules
bash attack_backconnect.sh
```

### 方法3：使用 PowerShell（需要安装 bash）
```powershell
# 在 PowerShell 中
bash attack_backconnect.sh
```

## 可用的攻击脚本列表

1. **attack_backconnect.sh** - 模拟后门端口连接
2. **attack_ngrok_ultra.sh** - 模拟 ngrok 域名连接
3. **attack_mining_pools_fast.sh** - 模拟挖矿池连接（快速版）
4. **attack_mining_pools_ultra.sh** - 模拟挖矿池连接（完整版）

## 快速测试所有脚本

```bash
# 一次性运行所有攻击脚本
bash attack_backconnect.sh && \
bash attack_ngrok_ultra.sh && \
bash attack_mining_pools_fast.sh
```

## 注意事项

- 这些脚本仅用于安全测试和EDR规则验证
- 确保系统已安装必要的工具：`nc` (netcat), `curl`, `host`
- 如果缺少工具，脚本会自动使用备用方法
- 运行后检查 EDR 系统日志以确认检测
